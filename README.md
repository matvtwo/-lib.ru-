Проект представляет собой серверное веб-приложение, реализующее интеллектуальный HTTP-прокси для исторического литературного архива lib.ru с последующей трансформацией и адаптацией гипертекстового контента под современные пользовательские интерфейсы чтения.

В рамках проекта реализован слой промежуточной обработки HTTP-запросов, который динамически маршрутизирует обращения к различным доменным зонам экосистемы lib.ru (lib.ru, lit.lib.ru, az.lib.ru), корректно разрешая относительные и абсолютные ссылки, восстанавливая иерархию каталогов и предотвращая уход пользователя на внешние ресурсы.

Особое внимание уделено обработке и нормализации текстовых данных, представленных в устаревших кодировках (Windows-1251, KOI8-R), с последующим приведением их к унифицированному UTF-8 представлению. Это обеспечивает корректную визуализацию контента в современных браузерах и упрощает дальнейшую работу с текстом.

Для страниц литературных произведений реализован специализированный режим «читалки», в котором HTML-структура оригинального документа преобразуется в поток чистого текста с сохранением логической структуры, удобной навигации, пагинации и пользовательского управления чтением.

Для каталогов авторов реализована клиентская интерактивная фильтрация, позволяющая выполнять локальный поиск по именам авторов без обращения к серверу, на основе анализа DOM-структуры страницы.

Архитектура проекта демонстрирует практическое применение принципов:
– HTTP-прокси и reverse-proxy,
– серверной трансформации HTML,
– работы с различными кодировками текста,
– DOM-манипуляций на стороне клиента,
– адаптации легаси-контента под современные UX-паттерны.

Проект может рассматриваться как пример интеграции исторических цифровых архивов в современные веб-системы чтения и анализа текста.

Используемые технологии:

Описание проекта

Проект представляет собой серверное веб-приложение, реализующее интеллектуальный HTTP-прокси для исторического литературного архива lib.ru, с последующей серверной трансформацией и адаптацией гипертекстового контента под современные пользовательские интерфейсы чтения.

В рамках проекта реализован промежуточный слой обработки HTTP-запросов, который динамически маршрутизирует обращения к различным доменным зонам экосистемы lib.ru (lib.ru, lit.lib.ru, az.lib.ru), корректно разрешая относительные и абсолютные ссылки, восстанавливая иерархию каталогов и предотвращая уход пользователя на внешние ресурсы.

Обработка и нормализация данных

Особое внимание уделено обработке текстовых данных, представленных в устаревших кодировках:

Windows-1251

KOI8-R

Получаемый HTML-контент декодируется и приводится к унифицированному UTF-8 представлению, что обеспечивает:

корректную визуализацию в современных браузерах;

возможность последующей машинной обработки текста;

совместимость с современными UX-интерфейсами.

Режим чтения (Reader Mode)

Для страниц литературных произведений реализован специализированный режим «читалки», в котором:

исходная HTML-структура документа преобразуется в чистый текст;

сохраняется логическая структура произведения;

реализована вертикальная навигация и постраничное перемещение;

добавлено пользовательское управление чтением (клавиатура, прокрутка).

Таким образом, легаси-контент, изначально не предназначенный для современных интерфейсов, адаптируется под комфортное длительное чтение.

Интерактивные каталоги авторов

Для страниц каталогов реализована клиентская интерактивная фильтрация авторов:

поиск выполняется локально в браузере, без сетевых запросов;

фильтрация основана на анализе DOM-структуры страницы;

обеспечивается мгновенный отклик интерфейса даже при большом количестве элементов.
